# Challenge 3 — Single-byte XOR Cipher

**Link:** https://cryptopals.com/sets/1/challenges/3

## What it does
A message was XORed with a single byte. Find the key and decrypt it.

## Key insight
Single-byte XOR has only 256 possible keys — small enough to try all of them. The problem is knowing which result is correct. The answer is English letter frequency: if you XOR with the right key, the output will look like English text (lots of spaces, 'e', 't', 'a', etc.). Score each attempt and pick the highest score.

The scoring function is the heart of this: common letters add points, non-printable bytes subtract points. It doesn't need to be perfect — it just needs to rank the correct answer above 255 wrong ones.

## Why it matters
`crack_single_byte_xor` becomes the most reused function in the entire project. It's used directly in ch04, ch06, and ch20. Getting the scoring right here pays off across everything.

## Functions
- `score_english()` — frequency-based scoring of how English-like a buffer looks
- `single_byte_xor()` — XOR every byte with the same key byte
- `crack_single_byte_xor()` — try all 256 keys, return the one with the highest score
