# Challenge 3 — Single-byte XOR Cipher

**Link:** https://cryptopals.com/sets/1/challenges/3

## What it does
A message was encrypted with XOR using a single byte as key. Find the key and decrypt it.

## How it works
When you XOR every byte of a message with the same key byte, you get ciphertext. Since a byte only has 256 possible values (0x00 to 0xFF), we can try all of them. The trick is knowing which result is the correct one — we use english letter frequency to score each attempt. The text with the highest score (most common english letters like 'e', 't', 'a', and spaces) is probably the real message.

## Strategy
1. Decode the hex input to bytes (reusing `hex_to_bytes` from ch01)
2. For each possible key (0 to 255):
   - XOR all bytes with that key
   - Score the result by how much it looks like english
3. The key with the highest score wins

## Functions
- `score_english()` — scores text by letter frequency (common letters add points, weird bytes subtract)
- `single_byte_xor()` — XOR every byte with the same key
- `crack_single_byte_xor()` — tries all 256 keys, returns the best one

## Reuses from
- `hex_to_bytes()` from ch01 — to decode the hex input in the test

## Summary
This is the first actual crypto attack in the challenges. It works because single-byte XOR has a tiny keyspace (only 256 options) and english text has predictable patterns. The frequency analysis technique used here is one of the oldest methods in cryptanalysis — it's been used to break ciphers since the 9th century. The scoring function and cracking logic from this challenge get reused in ch04 and ch06.
