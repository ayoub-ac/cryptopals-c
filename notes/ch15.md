# Challenge 15 — PKCS#7 padding validation

**Link:** https://cryptopals.com/sets/2/challenges/15

## What it does
Take a buffer with PKCS#7 padding and validate that the padding is correct. If it's bad, return an error.

## How it works
Read the last byte (N). Check that the last N bytes all have value N. If they do, strip them. If not, error.

## Valid examples
- `"ICE ICE BABY\x04\x04\x04\x04"` — last byte is 4, last 4 bytes are all 4. Valid.
- `"AAAAAAAAAAAAAAAA\x10\x10...\x10"` — full block of padding (16 bytes of 0x10). Valid.

## Invalid examples
- `"ICE ICE BABY\x05\x05\x05\x05"` — last byte says 5 but the data doesn't have 5 bytes of padding at position.
- `"ICE ICE BABY\x01\x02\x03\x04"` — last byte says 4 but the 4th-to-last is 0x01, not 0x04.

## Functions
- `pkcs7_validate_and_strip()` — validates and copies data without padding to output

## Reuses from
- Same concept as `pkcs7_unpad()` from ch09, but returns a new buffer instead of just a length

## Summary
This is a small but important building block. The next challenge (ch16) uses this for the bitflipping attack, and ch17 (padding oracle) depends entirely on whether the padding is valid or not.
