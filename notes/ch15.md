# Challenge 15 — PKCS#7 Padding Validation

**Link:** https://cryptopals.com/sets/2/challenges/15

## What it does
Validate that a buffer has correct PKCS#7 padding and strip it. Return an error if the padding is malformed.

## Key insight
Validation must be strict. Read the last byte (N), then verify that the last N bytes all have value N. Any deviation — wrong values, N=0, N larger than the block size — is invalid padding.

Valid:   `"ICE ICE BABY\x04\x04\x04\x04"` → last byte is 4, last 4 bytes are 0x04 ✓
Invalid: `"ICE ICE BABY\x05\x05\x05\x05"` → last byte says 5, but only 4 bytes match ✗
Invalid: `"ICE ICE BABY\x01\x02\x03\x04"` → last byte says 4, but 4th-to-last is 0x01 ✗

## Why strictness matters
This is not just about correctness — it's a security requirement. In ch17 (padding oracle), the attack works precisely because the server leaks whether padding is valid or invalid. A server that throws different errors for "bad padding" vs "wrong MAC" can be exploited to decrypt arbitrary ciphertexts one byte at a time. Strict, uniform error handling is part of the defense.

## Functions
- `pkcs7_validate_and_strip()` — validates padding and returns data without it, -1 on error
