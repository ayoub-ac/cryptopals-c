# Challenge 1 — Hex to Base64

**Link:** https://cryptopals.com/sets/1/challenges/1

## What it does
Convert a hex-encoded string into base64.

## How it works
Hex uses two characters per byte — the upper nibble (4 bits) and the lower nibble. Base64 groups bytes into 6-bit chunks, producing 4 output characters for every 3 input bytes. Neither is encryption, just different ways to represent the same data.

## Strategy
Go hex → raw bytes → base64. Never try to convert hex to base64 directly — always pass through raw bytes as the intermediate step.

## Functions
- `hex_to_bytes()` — decodes hex string to byte array, two chars at a time
- `bytes_to_base64()` — encodes raw bytes into base64 string
- `hex_to_base64()` — chains the two above

## Reuses from
This is the first challenge, no dependencies.

## Summary
This challenge introduces the two most common byte encodings: hexadecimal and base64. Hex represents each byte with 2 characters (one per nibble), while base64 packs 3 bytes into 4 characters using 6 bits each. The important thing is that both are just representations of the same raw bytes — they don't encrypt or hide anything. In cryptography we always work with raw bytes and only use hex/base64 for displaying results.
