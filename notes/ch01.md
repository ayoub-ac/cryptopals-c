# Challenge 1 — Hex to Base64

**Link:** https://cryptopals.com/sets/1/challenges/1

## What it does
Convert a hex-encoded string to base64.

## Key insight
Both hex and base64 are just different ways to write the same raw bytes — neither encrypts anything. The right mental model is: hex → raw bytes → base64. You never convert hex directly to base64; raw bytes are always the common language in the middle.

Hex gives 2 characters per byte (one per nibble). Base64 packs 3 bytes into 4 characters using 6-bit groups. So 3 input bytes → 4 base64 characters.

## Why it matters
Every challenge in this project passes data around as hex or base64. Getting these conversions right from the start means everything downstream just works. `hex_to_bytes` and `base64_to_bytes` get reused constantly.

## Functions
- `hex_to_bytes()` — two hex chars → one byte, repeated
- `bytes_to_base64()` — three bytes → four base64 chars, with `=` padding at the end
- `hex_to_base64()` — chains both above
