# Challenge 9 — PKCS#7 Padding

**Link:** https://cryptopals.com/sets/2/challenges/9

## What it does
Pad a buffer to a multiple of the block size using PKCS#7, then validate and remove it.

## Key insight
Block ciphers need input in fixed-size chunks. PKCS#7 solves this cleanly: if you need N bytes of padding, write N bytes each with value N. For example, 4 bytes of padding → `\x04\x04\x04\x04`. This way the receiver always knows how many padding bytes to strip — just read the last byte.

The edge case: if the data is already a multiple of the block size, you still add a full block of padding (`\x10` × 16 for 16-byte blocks). Without this rule, a message ending in `\x04\x04\x04\x04` would be ambiguous — is that data or padding?

Validation is strict: check that the last N bytes all have value N. If not, the data was tampered with or corrupted. This strictness becomes critical in ch17, where a server leaking "valid/invalid padding" is enough to decrypt the entire message.

## Functions
- `pkcs7_pad()` — add padding to reach the next block boundary
- `pkcs7_unpad()` — validate padding and strip it, returns -1 if invalid
