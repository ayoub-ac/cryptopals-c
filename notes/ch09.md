# Challenge 9 — PKCS#7 Padding

**Link:** https://cryptopals.com/sets/2/challenges/9

## What it does
Add padding to make data a multiple of the block size. Block ciphers like AES work with fixed-size blocks (16 bytes), so if your data isn't the right length you need to pad it.

## How it works
PKCS#7 padding is simple: if you need N bytes of padding, add N bytes each with value N. For example, if you need 4 bytes of padding, you add `\x04\x04\x04\x04`. If the data is already a multiple of the block size, you add a full block of padding (so the receiver always knows where the data ends).

## Strategy
- Pad: calculate how many bytes are missing (`block_size - len % block_size`), fill with that value
- Unpad: read the last byte to know how many padding bytes there are, verify they all match, remove them

## Functions
- `pkcs7_pad()` — add padding bytes
- `pkcs7_unpad()` — validate and remove padding, returns -1 if padding is wrong

## Reuses from
Nothing, this is independent.

## Summary
PKCS#7 is the standard padding scheme used with block ciphers. The padding value tells you how many bytes were added, so removing it is straightforward. Validating the padding is important — if the padding bytes don't all match, something is wrong (either corrupted data or a tampering attempt). This validation becomes critical in ch17 where a padding oracle attack exploits servers that leak whether padding is valid.
