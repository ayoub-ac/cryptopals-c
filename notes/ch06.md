# Challenge 6 — Break Repeating-key XOR

**Link:** https://cryptopals.com/sets/1/challenges/6

## What it does
A file was encrypted with repeating-key XOR (Vigenere). Find the key length, recover the key byte by byte, and decrypt the message.

## Step 1 — Find the key length using Hamming distance

Hamming distance between two byte buffers = number of bits that differ. Example:
```
"this is a test" vs "wokka wokka!!!" → 37 bits differ
```

The key observation: if you take two consecutive blocks of ciphertext of length `L` and XOR them, you get `plain_A XOR plain_B` (the keystream cancels). If `L` is the correct key length, those two plaintexts are both English — low bit-entropy. If `L` is wrong, you're XORing misaligned keystream bytes, producing something closer to random — high bit-entropy.

So we try key sizes 2 to 40, compute the normalized Hamming distance between the first few block pairs, and pick the key size with the lowest score:

```
normalized_distance = hamming(block_0, block_1) / key_size
```

We average over multiple block pairs to reduce noise.

## Step 2 — Transpose and crack each column

Once we know the key length (say 29), the ciphertext was XORed like this:
```
cipher[0]  = plain[0]  XOR key[0]
cipher[1]  = plain[1]  XOR key[1]
...
cipher[28] = plain[28] XOR key[28]
cipher[29] = plain[29] XOR key[0]   ← key wraps around
cipher[30] = plain[30] XOR key[1]
...
```

So bytes at positions 0, 29, 58, 87... were all XORed with the same byte `key[0]`. That column is exactly a single-byte XOR problem — solved with `crack_single_byte_xor` from ch03. Repeat for each of the 29 key positions.

This is the same transposition idea used later in ch20 against CTR with reused nonce.

## Result
The recovered key was **"Terminator X: Bring the noise"** (29 bytes). The decrypted text is a rap lyric, confirming the attack worked.

## Why it matters
This is the most technically involved challenge in Set 1. It chains three separate techniques: Hamming distance to guess key size, transposition to isolate individual key bytes, and frequency analysis to crack each one. Each technique is independently useless; together they break a cipher that looks much stronger than single-byte XOR.

## Functions
- `hamming_distance()` — count differing bits between two buffers (popcount of XOR)
- `guess_keysize()` — try sizes 2–40, return lowest normalized Hamming distance
- `break_repeating_key_xor()` — full pipeline: guess size → transpose → crack per column → decrypt
