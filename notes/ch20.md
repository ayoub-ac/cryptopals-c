# Challenge 20 — Break Fixed-Nonce CTR Statistically

**Link:** https://cryptopals.com/sets/3/challenges/20

## What it does
60 plaintexts (rap lyrics) all encrypted with AES-128-CTR using the same key and nonce=0. Recover the keystream and decrypt all of them statistically, with no manual guessing.

## Root cause: nonce reuse

Same `(key, nonce)` → same keystream for every message. So for every position `col`:
```
cipher_0[col] = plain_0[col] XOR keystream[col]
cipher_1[col] = plain_1[col] XOR keystream[col]
...
cipher_59[col] = plain_59[col] XOR keystream[col]
```

All 60 bytes at column `col` were XORed with the same single byte `keystream[col]`. That is exactly a single-byte XOR problem — the same one solved in ch03.

## The transposition trick (same as ch06)

Truncate all ciphertexts to the length of the shortest one (so every column is fully populated). Then transpose: instead of looking at messages horizontally, look at columns vertically.

```
Original view (rows = messages):
  cipher_0: [b00 b01 b02 b03 ...]
  cipher_1: [b10 b11 b12 b13 ...]
  cipher_2: [b20 b21 b22 b23 ...]

Transposed (columns = same keystream byte):
  col 0: [b00 b10 b20 ...]  ← all XORed with keystream[0]
  col 1: [b01 b11 b21 ...]  ← all XORed with keystream[1]
```

Each column is now a single-byte XOR cipher with 60 samples. Run `crack_single_byte_xor` on each → recover one keystream byte. 60 samples is enough for English frequency scoring to work reliably.

## Limitation

The first column (position 0) is often wrong. Capital letters at the start of sentences skew the frequency distribution — the scorer may prefer a lowercase shift over the correct uppercase one. This is a known limitation of statistical attacks on very short columns and explains why `test_ch20.cpp` validates from byte 1 onwards.

## Functions
- `load_plaintexts()` — read base64 lines from file, decode to raw bytes
- `encrypt_all_fixed_nonce()` — encrypt all plaintexts with same key and nonce=0
- `recover_keystream_stat()` — truncate to min length → transpose → `crack_single_byte_xor` per column
