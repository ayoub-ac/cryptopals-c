# Challenge 13 — ECB cut-and-paste

**Link:** https://cryptopals.com/sets/2/challenges/13

## What it does
A server creates user profiles like "email=X&uid=10&role=user" and encrypts them with AES-128-ECB. We forge a ciphertext that decrypts to "role=admin".

## How it works
ECB encrypts each 16-byte block independently. If we can control where text lands in each block, we can swap blocks from different ciphertexts to build a new valid profile.

## Strategy
1. Choose an email so that "role=" lands at the end of block 1 (byte 32), pushing "user" into block 2.
2. Craft another email so that "admin" + PKCS#7 padding starts at a block boundary.
3. Take blocks 0-1 from the first ciphertext and the "admin" block from the second.
4. The server decrypts the combined blocks and sees "role=admin".

## Block alignment
```
Email "AAAA@AAAA.com" gives:
  block 0: "email=AAAA@AAAA."
  block 1: "com&uid=10&role="
  block 2: "user" + padding

We replace block 2 with an "admin" block from a crafted email.
```

## Functions
- `kv_parse()` — parses "key=val&key=val" into a struct
- `profile_for()` — creates profile string, strips & and = from email
- `encrypt_profile()` / `decrypt_profile()` — encrypt/decrypt with server key
- `ecb_cut_and_paste()` — the attack: crafts two ciphertexts and swaps blocks

## Reuses from
- `aes_128_ecb_encrypt()` / `aes_128_ecb_decrypt()` from ch07

## Summary
This shows why ECB is dangerous with structured data. Because blocks are independent, an attacker can rearrange them to change the meaning of the decrypted data. The server can't tell the difference between a real profile and a forged one.
