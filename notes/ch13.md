# Challenge 13 — ECB Cut-and-Paste

**Link:** https://cryptopals.com/sets/2/challenges/13

## What it does
A server builds user profile strings (`email=X&uid=10&role=user`) and encrypts them with AES-128-ECB. Forge a ciphertext that decrypts to `role=admin` — without knowing the key.

## Why ECB makes this possible

ECB encrypts each 16-byte block independently with no interaction between blocks. This means ciphertext blocks are portable: you can take block 2 from one ciphertext and block 0 from another, concatenate them, and the server will decrypt each block correctly. There's no chaining, no integrity check — nothing tying the blocks together.

## The attack in two queries

**Query 1 — isolate "role=" at the end of a block.**
The oracle strips `&` and `=` from the email input. Choose an email length so that `role=` falls exactly at the end of block 1 and `user` starts at block 2:

```
email = "AAAA@AAAA.com"  (13 chars)

block 0: [email=AAAA@AAAA.]   16 bytes
block 1: [com&uid=10&role=]   16 bytes  ← ends with "role="
block 2: [user\x0c\x0c\x0c\x0c\x0c\x0c\x0c\x0c\x0c\x0c\x0c\x0c]  ← "user" + padding
```

Keep blocks 0 and 1 from this ciphertext.

**Query 2 — get "admin" with valid padding in its own block.**
Craft an email where `admin` + PKCS#7 padding (`\x0b` × 11) starts at a block boundary:

```
email = "AAAAAAAAAA" + "admin\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b"

block 0: [email=AAAAAAAAAA]
block 1: [admin\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b]  ← standalone block
```

Keep block 1 from this ciphertext.

**Forge:** concatenate blocks 0+1 from query 1 with block 1 from query 2. The server decrypts:
```
"email=AAAA@AAAA.com&uid=10&role=admin"  ← role=admin ✓
```

## Why it matters
This attack requires only two oracle queries and no key knowledge. ECB-encrypted tokens used for access control are trivially forgeable if an attacker can query the encryption oracle. The fix is using a mode with authentication (AES-GCM) or adding a MAC over the plaintext.

## Functions
- `kv_parse()` — parse `key=val&key=val` format
- `profile_for()` — build profile, strip `&` and `=` from email
- `encrypt_profile()` / `decrypt_profile()` — oracle encrypt/decrypt
- `ecb_cut_and_paste()` — two oracle queries + block reassembly
