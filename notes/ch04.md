# Challenge 4 — Detect Single-Character XOR

**Link:** https://cryptopals.com/sets/1/challenges/4

## What it does
A file contains 326 hex-encoded strings. One of them was encrypted with single-byte XOR. Find which one.

## How it works
This builds directly on ch03. Instead of cracking one string, we crack all 326 and see which one produces the most english-like result. The one with the highest score is the encrypted line.

## Strategy
1. Read each line from the file
2. Decode hex to bytes (using `hex_to_bytes` from ch01)
3. Run `crack_single_byte_xor` from ch03 on each line
4. Keep track of the line with the highest score
5. That's the answer

## Functions
- `detect_single_byte_xor()` — reads a file of hex strings, cracks each one, returns the best

## Reuses from
- `hex_to_bytes()` from ch01 — to decode each hex line
- `crack_single_byte_xor()` from ch03 — to try all 256 keys on each line

## Summary
This challenge shows the value of reusing code. We already had everything we needed from ch01 and ch03 — the only new code is reading lines from a file and comparing scores. This is a common pattern in cryptanalysis: apply an existing attack to many candidates and filter the results. The detected message is "Now that the party is jumping", encrypted with key 0x35 ('5').
