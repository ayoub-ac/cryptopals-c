# Challenge 11 — ECB/CBC Detection Oracle

**Link:** https://cryptopals.com/sets/2/challenges/11

## What it does
An oracle encrypts our input with a random key, randomly choosing ECB or CBC. We have to detect which mode was used without knowing the key.

## How it works
The oracle adds 5-10 random bytes before and after our input, then encrypts with either ECB or CBC (50/50 chance). The key is random and different each time. We can't decrypt, but we can detect the mode.

## Strategy
Feed the oracle 48 bytes of the same value (like 'AAAAAA...'). Even with random prefix/suffix, there will be at least two identical 16-byte blocks in the middle. If ECB was used, those identical blocks produce identical ciphertext. If CBC, the chaining makes them different.

## Functions
- `encryption_oracle()` — encrypts with random key and random ECB/CBC choice
- `detect_ecb_or_cbc()` — checks for repeated blocks (reuses `count_repeated_blocks` from ch08)

## Reuses from
- `aes_128_ecb_encrypt()` from ch07
- `aes_128_cbc_encrypt()` from ch10
- `count_repeated_blocks()` from ch08

## Summary
This challenge shows that ECB is detectable even when we don't know the key. By choosing our input carefully (repeated bytes), we force repeated plaintext blocks. ECB encrypts them identically, CBC doesn't. This is the basis for the "chosen plaintext" attack model used in the next challenges.
