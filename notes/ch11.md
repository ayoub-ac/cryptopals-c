# Challenge 11 — ECB/CBC Detection Oracle

**Link:** https://cryptopals.com/sets/2/challenges/11

## What it does
An oracle encrypts our input with a random key, choosing ECB or CBC randomly (50/50). Detect which mode was used without knowing the key.

## Key insight
Force the oracle to reveal itself by sending input designed to produce identical plaintext blocks. Send 48 identical bytes — even with the oracle's random 5–10 byte prefix and suffix, there will be at least two full identical 16-byte blocks in the middle.

With ECB: identical plaintext blocks → identical ciphertext blocks → detectable with `count_repeated_blocks`.
With CBC: the chaining XOR makes even identical plaintext blocks produce different ciphertext.

The oracle has no defense against this because ECB's block-independence is structural — it's not a bug in the implementation, it's how the mode works.

## Why it matters
This is the first "chosen plaintext" challenge — we don't just observe ciphertext, we control the input and study how the output changes. This adversarial model is used in all the attacks that follow (ch12, ch13, ch14, ch16).

## Functions
- `encryption_oracle()` — random key, random ECB/CBC, random prefix/suffix
- `detect_ecb_or_cbc()` — send crafted input, check for repeated blocks
