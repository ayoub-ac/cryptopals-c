# Challenge 7 — AES in ECB Mode

**Link:** https://cryptopals.com/sets/1/challenges/7

## What it does
Decrypt a base64-encoded file encrypted with AES-128-ECB. Key is "YELLOW SUBMARINE".

## Key insight
AES is a block cipher: it takes exactly 16 bytes of plaintext and a key, and produces 16 bytes of ciphertext. ECB (Electronic Codebook) is the simplest mode — it applies AES independently to each 16-byte block with no interaction between blocks.

We use OpenSSL's EVP API instead of implementing AES ourselves. EVP handles the actual math; we just provide the key, mode, and data. This is the right call — implementing AES correctly from scratch is error-prone and adds no learning value for this project.

## Why it matters
`aes_128_ecb_encrypt` and `aes_128_ecb_decrypt` become the lowest-level building block for everything that follows. CBC (ch10) is built on top of raw ECB blocks. CTR (ch18) also uses raw ECB. Getting the OpenSSL EVP interface right here means all subsequent modes work correctly.

ECB's weakness — identical plaintext blocks produce identical ciphertext blocks — is exploited in ch08, ch11, ch12, ch13, and ch14.

## Functions
- `aes_128_ecb_encrypt()` — encrypt with AES-128-ECB via OpenSSL EVP
- `aes_128_ecb_decrypt()` — decrypt with AES-128-ECB via OpenSSL EVP
