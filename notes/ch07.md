# Challenge 7 — AES in ECB Mode

**Link:** https://cryptopals.com/sets/1/challenges/7

## What it does
Decrypt a base64-encoded file that was encrypted with AES-128 in ECB mode. The key is "YELLOW SUBMARINE".

## How it works
AES (Advanced Encryption Standard) is a block cipher that works with 16-byte blocks. ECB (Electronic Codebook) is the simplest mode: each block is encrypted/decrypted independently using the same key. We don't implement AES ourselves — we use OpenSSL's EVP interface which handles the math internally.

## Strategy
1. Read and decode the base64 file (using `read_base64_file` from ch01)
2. Decrypt with AES-128-ECB using OpenSSL's EVP API
3. The key is "YELLOW SUBMARINE" (exactly 16 bytes)

## Functions
- `aes_128_ecb_decrypt()` — decrypt using OpenSSL EVP
- `aes_128_ecb_encrypt()` — encrypt using OpenSSL EVP (needed for later challenges)

## Reuses from
- `read_base64_file()` from ch01 — to read and decode the input file

## Summary
AES-128-ECB encrypts each 16-byte block independently with the same key. This is simple but has a weakness: identical plaintext blocks produce identical ciphertext blocks, so patterns in the data are visible even after encryption. OpenSSL handles the actual AES math through its EVP interface — we just call the functions and pass the key.
