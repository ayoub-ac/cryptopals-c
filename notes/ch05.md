# Challenge 5 — Repeating-key XOR

**Link:** https://cryptopals.com/sets/1/challenges/5

## What it does
Encrypt a message by XORing each byte with the corresponding byte of a repeating key (Vigenere cipher).

## Key insight
The modulo operator makes the key cycle: `key[i % key_len]`. When `i` reaches the end of the key, it wraps back to position 0. This is the entire algorithm.

Since XOR is its own inverse, encryption and decryption are identical operations — XOR with the key both ways. That means `repeating_key_xor` works for both directions.

## Why it matters
This cipher looks stronger than single-byte XOR because the key is longer, but ch06 will break it completely. The weakness is that every key_len-th byte was XORed with the same key byte — so if you know the key length, you can split the ciphertext into columns and reduce it back to multiple single-byte XOR problems.

## Functions
- `repeating_key_xor()` — XOR data with a cycling key, byte by byte
