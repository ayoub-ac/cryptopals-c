# Challenge 5 — Repeating-key XOR

**Link:** https://cryptopals.com/sets/1/challenges/5

## What it does
Encrypt a message using a multi-byte key that repeats (Vigenere cipher). The key "ICE" cycles: byte 0 with 'I', byte 1 with 'C', byte 2 with 'E', byte 3 with 'I' again, etc.

## How it works
Instead of XORing every byte with the same single byte (ch03), we cycle through a key. The modulo operator (`%`) makes the key repeat: `key[i % key_len]` wraps around when we reach the end of the key.

## Strategy
For each byte in the input, XOR it with `key[i % key_len]`. That's it — one line of actual logic.

## Functions
- `repeating_key_xor()` — XOR data with a cycling key

## Reuses from
- `bytes_to_hex()` from ch02 — to convert the encrypted result to hex in the test

## Summary
This is the Vigenere cipher, one of the oldest encryption methods. It's stronger than single-byte XOR because the key is longer, but it's still breakable if you can guess the key length (which is what ch06 does). The encrypt and decrypt operations are identical — just XOR with the same key again. The modulo trick (`i % key_len`) is a common pattern in C for cycling through arrays.
