# Challenge 24 — Create the MT19937 stream cipher and break it

**Link:** https://cryptopals.com/sets/3/challenges/24

## What it does
Build a stream cipher using MT19937 as the keystream, then break it by brute force on a 16-bit key.

## How it works
The "key" is a 16-bit seed for MT19937. To encrypt, seed MT and generate enough output to XOR with the plaintext. Each 32-bit MT output gives 4 bytes of keystream.

## Strategy
**Encryption:**
1. Seed MT with the key
2. For each chunk of 4 bytes, extract one MT value
3. XOR each byte of the chunk with one byte of the MT value
4. Same operation works for decryption (XOR is reversible)

**Cracking:**
1. The cipher uses a 16-bit key → only 65536 possible keys
2. We know part of the plaintext (e.g. a known suffix)
3. Try every key, decrypt, check if the known text appears at the right offset
4. Stop at the first match

## Functions
- `mt_stream_cipher(key, in, len, out)` — encrypt or decrypt with 16-bit key
- `crack_mt_stream(cipher, len, known, offset, key_out)` — brute force the key

## Reuses from
- `mt_seed()` and `mt_extract()` from ch21

## Summary
A 16-bit key is way too small for any cipher. Even though the underlying PRNG (MT19937) has a huge state, using a tiny seed makes brute force trivial. Real ciphers need keys of at least 128 bits.
