# Challenge 24 — MT19937 Stream Cipher

**Link:** https://cryptopals.com/sets/3/challenges/24

## What it does
Build a stream cipher using MT19937 as the keystream generator, keyed with a 16-bit seed. Then break it by brute force.

## How it works
Seed MT19937 with the 16-bit key. For each 4 bytes of input, extract one 32-bit MT value and XOR it byte by byte with the plaintext. Decryption is the same operation (XOR is its own inverse).

## Key insight
A 16-bit key has only 65,536 possible values. Even though MT19937's internal state is 624 words (19,937 bits), the seed collapses all that potential into a tiny keyspace. Brute forcing every possible key takes milliseconds.

The attack: we know part of the plaintext (a known suffix — like a fixed string appended to user data). For each candidate key, decrypt and check if the known plaintext appears at the expected offset. First match is the key.

```
for k = 0 to 65535:
    mt_stream_cipher(k, cipher, len, buf)
    if buf[offset..offset+known_len] == known_plaintext:
        found key = k
```

## Why it matters
This demonstrates that the size of the PRNG's internal state is irrelevant if the seed is small. A cipher is only as strong as its key. Real stream ciphers (ChaCha20, AES-CTR) use 128 or 256-bit keys — brute force is computationally infeasible. 16 bits is a toy.

## Functions
- `mt_stream_cipher()` — encrypt/decrypt with MT19937 keystream, 16-bit key
- `crack_mt_stream()` — brute force all 65,536 keys using known plaintext
