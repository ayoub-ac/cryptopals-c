# Challenge 21 — Implement the MT19937 Mersenne Twister RNG

**Link:** https://cryptopals.com/sets/3/challenges/21

## What it does
Implements the MT19937 pseudorandom number generator. Given a 32-bit seed, it produces a deterministic sequence of 32-bit outputs. Same seed = same sequence every time.

## How it works
MT19937 keeps an internal state of 624 uint32_t values. It has three phases:

1. **Seed**: Initialize the 624 entries from the seed using a recurrence relation.
2. **Twist**: Every 624 extractions, regenerate the entire state array. Each new value mixes two adjacent entries and XORs with a constant matrix.
3. **Temper**: Each output goes through a series of shifts and masks to improve the statistical distribution.

## Constants
| Name | Value | Purpose |
|------|-------|---------|
| n | 624 | state size |
| m | 397 | twist offset |
| a | 0x9908B0DF | matrix constant |
| upper_mask | 0x80000000 | bit 31 |
| lower_mask | 0x7FFFFFFF | bits 0-30 |
| f | 1812433253 | seed init multiplier |

## Tempering
```
y ^= (y >> 11)
y ^= (y << 7)  & 0x9D2C5680
y ^= (y << 15) & 0xEFC60000
y ^= (y >> 18)
```

## Functions
- `mt_seed()` — initialize state array from a 32-bit seed
- `mt_extract()` — return next random number, triggers twist if needed

## Why it matters for crypto
MT19937 is NOT cryptographically secure. The state is fully recoverable from 624 consecutive outputs (challenge 23). It's commonly used in games, simulations, and standard library `rand()`, but never for crypto.

## Summary
Straightforward implementation following the spec. The tricky part is getting all the constants and shifts right — one wrong bit and the outputs won't match reference values. I verified mine against the known outputs for seeds 0 and 1.
