# Challenge 21 — Implement MT19937 Mersenne Twister

**Link:** https://cryptopals.com/sets/3/challenges/21

## What it does
Implement the MT19937 pseudorandom number generator. Given a 32-bit seed it produces a deterministic sequence of 32-bit values.

## How it works
MT19937 maintains an internal state of 624 `uint32_t` values and works in three phases:

**Seed:** initialize the 624 entries using a recurrence relation starting from the seed value.

**Twist:** every 624 extractions, regenerate the entire state array. Each new value is computed by mixing two entries from the current state and XORing with a constant (`MATRIX_A = 0x9908B0DF`).

**Temper:** before returning each value, apply a series of shifts and masks to spread the bits more uniformly:
```c
y ^= (y >> 11)
y ^= (y << 7)  & 0x9D2C5680
y ^= (y << 15) & 0xEFC60000
y ^= (y >> 18)
```

## Key detail
The tempering transform is what ch23 will invert. It's there to improve statistical quality, but it's not a cryptographic operation — it's invertible, which means anyone with 624 outputs can reconstruct the full internal state.

## Why it matters
MT19937 is NOT cryptographically secure. It's widely used in game engines, scientific simulations, and standard library `rand()` — but never for security-sensitive applications. The state is fully recoverable from output (ch23), and the seed space is often small and predictable (ch22). I verified the implementation against known reference outputs for seeds 0 and 1.

## Functions
- `mt_seed()` — initialize state from a 32-bit seed
- `mt_extract()` — return the next number, triggering a twist if the state is exhausted
