# Challenge 8 — Detect AES in ECB Mode

**Link:** https://cryptopals.com/sets/1/challenges/8

## What it does
A file has many hex-encoded ciphertexts. One of them was encrypted with AES in ECB mode. Find which one.

## How it works
ECB encrypts each 16-byte block independently. If the same 16-byte block of plaintext appears more than once, the ciphertext blocks will also be identical. Real data often has repeated patterns (like HTTP headers or structured data), so ECB ciphertexts tend to have repeated blocks.

## Strategy
1. Read each hex line from the file
2. Decode to bytes
3. Count how many 16-byte blocks are repeated
4. The line with the most repeats is the ECB-encrypted one

## Functions
- `count_repeated_blocks()` — compares all pairs of 16-byte blocks using memcmp
- `detect_ecb_in_file()` — reads hex lines, counts repeats, returns the best line

## Reuses from
- `hex_to_bytes()` from ch01 — to decode each line

## Summary
ECB's weakness is that identical plaintext blocks produce identical ciphertext blocks. This makes it easy to detect — just look for repeated 16-byte blocks in the ciphertext. In practice, ECB should never be used for real data because these patterns leak information even without decrypting.
