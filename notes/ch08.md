# Challenge 8 — Detect AES in ECB Mode

**Link:** https://cryptopals.com/sets/1/challenges/8

## What it does
A file has many hex-encoded ciphertexts. One was encrypted with AES-ECB. Find which one.

## Key insight
ECB's fatal flaw: identical plaintext blocks → identical ciphertext blocks. Real data often has repeated structure (HTTP headers, fixed-format fields, padding). If any 16-byte block repeats in the ciphertext, it's almost certainly ECB.

The approach: split each ciphertext into 16-byte blocks and count how many are duplicates. The line with the most repeated blocks is the ECB-encrypted one. Every other mode (CBC, CTR) would produce completely different ciphertext blocks even for identical plaintext.

## Why it matters
This detection technique reappears in ch11 (ECB/CBC oracle) and ch13/14 (chosen-plaintext attacks). Recognizing ECB from its block-repetition pattern is fundamental to understanding why it's insecure.

## Functions
- `count_repeated_blocks()` — compare all pairs of 16-byte blocks, count duplicates
- `detect_ecb_in_file()` — find the line with the most repeated blocks
