# Challenge 2 — Fixed XOR

**Link:** https://cryptopals.com/sets/1/challenges/2

## What it does
XOR two equal-length hex buffers and produce the result.

## How it works
XOR compares two bits: if they are different the result is 1, if they are the same the result is 0. The key property is that XOR cancels itself: a ^ b ^ b = a. This is why XOR is used in encryption — you can encrypt and decrypt with the same operation.

## Strategy
Decode both hex strings to bytes (reusing `hex_to_bytes` from ch01), XOR them byte by byte, then encode the result back to hex.

## Functions
- `fixed_xor()` — XOR two byte buffers of the same length
- `bytes_to_hex()` — convert raw bytes to hex string (inverse of `hex_to_bytes` from ch01)

## Reuses from
- `hex_to_bytes()` from ch01 — to decode the hex input

## Summary
XOR is the fundamental operation in symmetric cryptography. It compares bits one by one: different bits give 1, same bits give 0. The most important property is that it cancels itself — if you XOR something twice with the same key, you get the original back (a ^ b ^ b = a). This makes it perfect for encryption because the same operation encrypts and decrypts. Every cipher we'll see in the next challenges uses XOR in some way.
