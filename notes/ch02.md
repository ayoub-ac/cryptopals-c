# Challenge 2 — Fixed XOR

**Link:** https://cryptopals.com/sets/1/challenges/2

## What it does
XOR two equal-length byte buffers together.

## Key insight
XOR is the fundamental operation in symmetric cryptography. It works bit by bit: same bits → 0, different bits → 1. The critical property is that it cancels itself: `a ^ b ^ b = a`. This means the same operation both encrypts and decrypts — XOR with the key to lock, XOR with the key again to unlock.

Every cipher in this project uses XOR in some form. Understanding it here makes everything else easier.

## Why it matters
`fixed_xor` is a building block used everywhere: CBC chaining, CTR keystream application, bitflipping attacks. Getting the byte-by-byte loop right once means it's reusable.

## Functions
- `fixed_xor()` — XOR two buffers byte by byte
- `bytes_to_hex()` — inverse of `hex_to_bytes` from ch01, used to display results
