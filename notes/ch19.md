# Challenge 19 — Break Fixed-Nonce CTR (Manual)

**Link:** https://cryptopals.com/sets/3/challenges/19

## What it does
Many plaintexts were each encrypted with AES-128-CTR using the same key and the same nonce. Recover the keystream and decrypt them.

## Key insight
Same nonce + same key = same keystream for every message. That means:
```
cipher_A[i] = plain_A[i] XOR keystream[i]
cipher_B[i] = plain_B[i] XOR keystream[i]
```

XOR the two ciphertexts: `cipher_A[i] XOR cipher_B[i] = plain_A[i] XOR plain_B[i]`. The keystream cancels out. You're left with plaintext XOR plaintext — the classical many-time pad problem.

For each position `i`, every ciphertext byte was XORed with the same keystream byte. That column is a single-byte XOR problem: try all 256 values for `keystream[i]`, score the resulting column for English frequency, take the best one.

## Approach in this implementation
The plaintexts are short English sentences, so the statistical signal per column is weak. We use manual frequency analysis supplemented by educated guessing — trying candidate keystream bytes and visually checking if the decrypted columns produce readable English. This is the "manual" part vs ch20's fully statistical approach.

## Why it matters
This is the many-time pad attack. One-time pads are theoretically unbreakable — but only if the key is truly random AND never reused. Reusing a nonce in CTR converts a strong stream cipher into a catastrophically weak one.

## Functions
- `encrypt_fixed_nonce()` — encrypt multiple plaintexts with same key and nonce=0
- `recover_keystream()` — column-by-column keystream recovery
