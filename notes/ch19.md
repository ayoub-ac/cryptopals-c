# Challenge 19 — Break fixed-nonce CTR

**Link:** https://cryptopals.com/sets/3/challenges/19

## What it does
40 plaintexts are all encrypted with AES-128-CTR using the same key AND the same nonce (0). Recover the keystream so we can decrypt all of them.

## Why it breaks
CTR works like a stream cipher: it generates a keystream from (key, nonce, counter) and XORs it with the plaintext. If you reuse the same nonce, all messages share the same keystream. That's catastrophic.

## Strategy
For each column position `i`, the byte at position `i` in every ciphertext was XORed with the same keystream byte. So if we collect `cipher[0][i], cipher[1][i], cipher[2][i]...` they form a buffer that was encrypted with a single-byte XOR.

That's exactly ch03! We can crack each column independently.

## Steps
1. Cut all ciphertexts to the shortest length
2. For each column, build the column buffer
3. Run `crack_single_byte_xor` on it -> recover one keystream byte
4. After all columns done, XOR the keystream with each ciphertext to decrypt

## Functions
- `encrypt_fixed_nonce()` - encrypts the 40 plaintexts with the same key and nonce=0
- `recover_keystream()` - column-by-column attack using single-byte XOR cracking

## Reuses from
- `aes_128_ctr()` from ch18
- `crack_single_byte_xor()` from ch03
- `base64_to_bytes()` from ch01

## Summary
Never reuse a nonce in CTR mode. The whole security depends on the keystream being unique per message. Reusing it turns CTR into a many-time pad, which is trivially breakable with frequency analysis on the columns.
