# Challenge 10 — Implement CBC Mode

**Link:** https://cryptopals.com/sets/2/challenges/10

## What it does
Implement AES-128-CBC encryption and decryption ourselves, using only raw AES-ECB for each block. Cryptopals specifically says not to use OpenSSL's built-in CBC.

## How it works
CBC (Cipher Block Chaining) adds a dependency between blocks. Before encrypting each plaintext block, it gets XORed with the previous ciphertext block. The first block uses an IV (initialization vector) since there's no previous block yet. This means identical plaintext blocks produce different ciphertext — fixing ECB's main weakness.

## Strategy
- **Decrypt**: for each 16-byte block, ECB decrypt it, then XOR with the previous ciphertext block (or IV for the first one). Finally remove PKCS#7 padding.
- **Encrypt**: pad the plaintext first, then for each block, XOR with the previous ciphertext (or IV), then ECB encrypt.

## Functions
- `aes_128_cbc_decrypt()` — CBC decryption using raw ECB + XOR
- `aes_128_cbc_encrypt()` — CBC encryption using XOR + raw ECB
- `ecb_block_decrypt/encrypt()` — raw AES-ECB for one block with no padding (static/internal)
- `xor_blocks()` — XOR two 16-byte blocks (static/internal)

## Reuses from
- `aes_128_ecb` functions from ch07 (via OpenSSL) — for raw block encryption
- `pkcs7_pad()` and `pkcs7_unpad()` from ch09 — for padding

## Summary
CBC mode chains blocks together by XORing each plaintext block with the previous ciphertext before encrypting. This means identical plaintext blocks produce different ciphertext, which is a big improvement over ECB. The IV is like a random starting point that makes the first block unique too. We built CBC on top of raw ECB blocks and XOR — the two building blocks are simple, but combining them correctly is where the challenge is.
