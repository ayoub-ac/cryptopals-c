# Challenge 12 — Byte-at-a-time ECB decryption (Simple)

**Link:** https://cryptopals.com/sets/2/challenges/12

## What it does
An oracle appends an unknown string to our input and encrypts with AES-128-ECB. We decrypt the unknown string without knowing the key.

## How it works
ECB encrypts each 16-byte block independently. Same plaintext block = same ciphertext block. We exploit this by controlling the input length.

## Strategy
1. **Discover block size**: Feed increasing bytes until the ciphertext grows. The jump = block size (16).
2. **Confirm ECB**: Repeated input blocks should produce repeated ciphertext blocks.
3. **Decrypt byte by byte**: For each unknown byte:
   - Send (block_size - 1) padding bytes so the target byte is at the end of a block
   - Get the reference ciphertext
   - Try all 256 possible values in that position
   - The one that matches the reference block is the correct byte
   - Move to the next byte

## Example
```
Block = 16 bytes. Unknown starts with "R..."

Round 1: send "AAAAAAAAAAAAAAA" (15 A's)
  Oracle encrypts: [AAAAAAAAAAAAAAAR] [est of unknown...]
  We get ciphertext block 0 as reference.

  Then try: [AAAAAAAAAAAAAAA\x00], [AAAAAAAAAAAAAAA\x01], ... [AAAAAAAAAAAAAAAR]
  When byte = 'R' (0x52), block 0 matches. First byte found!

Round 2: send "AAAAAAAAAAAAAA" (14 A's)
  Oracle encrypts: [AAAAAAAAAAAAAARo] [...]
  Try: [AAAAAAAAAAAAAARx] for all x. Match at 'o'. Second byte found!
```

## Functions
- `ecb_oracle()` — appends unknown string to input, encrypts with fixed key
- `discover_block_size()` — feeds increasing input to detect block size
- `decrypt_ecb_simple()` — the byte-by-byte attack

## Summary
This is the first "chosen plaintext" attack. We can't see the key but we control part of the input. Because ECB is deterministic, we can use that control to extract the secret one byte at a time. This is why ECB mode should never be used with attacker-controlled input.
