# Challenge 12 — Byte-at-a-time ECB Decryption (Simple)

**Link:** https://cryptopals.com/sets/2/challenges/12

## What it does
An oracle encrypts `input + secret` with AES-128-ECB using a fixed random key. We control `input` but know neither the key nor the secret. We recover the secret byte by byte.

## Phase 1 — Discover block size and secret length

Feed 1, 2, 3... bytes to the oracle until the ciphertext length jumps. The jump size is the block size. The number of bytes needed to trigger the jump (`i`) is the amount of padding that was filling the last block, which gives the secret length:

```
secret_len = base_cipher_len - i
```

## Phase 2 — Confirm ECB mode

Send 32 identical bytes. If block 0 and block 1 of the ciphertext are equal, the oracle uses ECB. Any other mode (CBC, CTR) would make them differ. Only proceed with the attack if ECB is confirmed.

## Phase 3 — Byte-at-a-time attack

This is the core of the challenge. For each byte `i` of the secret:

**a) Get the reference ciphertext.**
Send `pad_len = (block_size - 1 - i % block_size)` bytes of `'A'`. The oracle encrypts:
```
[AAAAAA...A | secret[0] secret[1] ... secret[i] | ...]
             ^--- byte i lands at the end of block `block_num`
```
Save the ciphertext of block `block_num` as the reference.

**b) Build the dictionary.**
Send `pad_len` bytes of `'A'` followed by the `i` already-known secret bytes, then a guess byte `b`:
```
[AAAAAA...A | secret[0] ... secret[i-1] | b]
```
Try `b` from 0 to 255. When block `block_num` of this ciphertext matches the reference, `b` is `secret[i]`.

**Block progression example (block_size = 16):**
```
i=0:  pad=15, block=0  [AAAAAAAAAAAAAAA ?]             → secret[0]
i=1:  pad=14, block=0  [AAAAAAAAAAAAAA s0 ?]           → secret[1]
...
i=15: pad=0,  block=0  [s0 s1 ... s14 ?]               → secret[15]
i=16: pad=15, block=1  [AAAAAAAAAAAAAAA][s0..s14 ?]    → secret[16]
i=17: pad=14, block=1  [AAAAAAAAAAAAAA ][s0..s15 ?]    → secret[17]
```

Every 16 bytes, `block_num` increments by 1 and the reference shifts to the next block.

## Why it works
ECB is deterministic: same 16-byte plaintext block → same 16-byte ciphertext block, always, regardless of key. By controlling all bytes in a block except one, we turn the oracle into a lookup table. At most 256 queries per byte.

## Cost
Up to `256 × secret_len` oracle calls. For a 138-byte secret that's at most ~35,000 calls.

## Functions
- `ecb_oracle()` — encrypts `input + secret` with a fixed random key
- `discover_block_size()` — feeds increasing input, returns the jump size
- `decrypt_ecb_simple()` — the full byte-by-byte attack loop
