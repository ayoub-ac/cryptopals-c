# Challenge 22 — Crack an MT19937 Seed

**Link:** https://cryptopals.com/sets/3/challenges/22

## What it does
A program seeds MT19937 with the current Unix timestamp and returns the first output. Given that output and approximately when it ran, recover the seed.

## Key insight
Using a timestamp as a seed is a tiny keyspace. Unix timestamps are seconds since 1970 — but we don't need to search all of history. If we know the output was generated "recently" (within the last few minutes), the search window is at most a few thousand candidates. Brute force from `now` backwards until MT19937 seeded with some candidate produces the observed output.

```
for candidate = now down to now - 2000:
    mt_seed(candidate)
    if mt_extract() == observed_output:
        found the seed
```

The search takes milliseconds even in C.

## Why it matters
Predictable seeds make the entire PRNG predictable. This applies beyond MT19937: any PRNG seeded with a timestamp, process ID, or other low-entropy value is vulnerable to this attack. Security-sensitive code must seed with output from a cryptographically secure source (like `/dev/urandom` on Linux or `BCryptGenRandom` on Windows).

## Functions
- `mt_seed_with_time()` — simulates the scenario: uses current time ± random offset as seed
- `crack_mt_seed()` — brute forces a window of candidate timestamps
