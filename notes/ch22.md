# Challenge 22 — Crack an MT19937 seed

**Link:** https://cryptopals.com/sets/3/challenges/22

## What it does
A program seeds MT19937 with the current Unix timestamp. We see only the first 32-bit output. We have to recover the seed.

## How it works
Brute force the timestamp. Since the seed is "now minus a few minutes", just try every timestamp going back until one matches.

## Strategy
1. Take the current time `now`
2. For each candidate from `now` down to `now - 2000`:
   - Seed MT19937 with the candidate
   - Extract one number
   - If it matches the output we have, that's the seed
3. Stop at the first match

## Functions
- `mt_seed_with_time()` — simulates the scenario: picks a time-based seed, returns the first output
- `crack_mt_seed(output, now)` — brute force the seed window

## Reuses from
- `mt_seed()` and `mt_extract()` from ch21

## Summary
Using a timestamp as a seed is a terrible idea. The space of possible seeds is tiny, so an attacker can find it in milliseconds. Use a cryptographically secure RNG instead.
