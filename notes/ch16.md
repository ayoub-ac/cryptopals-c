# Challenge 16 — CBC bitflipping attack

**Link:** https://cryptopals.com/sets/2/challenges/16

## What it does
A server wraps our input in a prefix/suffix and encrypts with CBC. It strips `;` and `=` so we can't inject ";admin=true;" directly. We modify the ciphertext to inject it anyway.

## How it works
In CBC, flipping a bit in ciphertext block N corrupts block N when decrypted, but flips the exact same bit in block N+1. We use this to change specific characters in the plaintext.

## Strategy
1. Send "XadminXtrueX" as input (X = placeholder for ; and =)
2. The server encrypts it. X lands at known positions.
3. Flip bits in the previous ciphertext block: `cipher[pos] ^= 'X' ^ ';'`
4. When decrypted, block N is garbage but block N+1 now has ";admin=true;"

## Why it works
CBC decrypt: `plaintext[N+1] = AES_decrypt(cipher[N+1]) ^ cipher[N]`

If we change `cipher[N]`, the XOR changes, so `plaintext[N+1]` changes at the same position. We know the original byte ('X') and the target (';'), so we XOR with `'X' ^ ';'` to flip it.

## Functions
- `bitflip_encrypt()` — wraps input in prefix/suffix, strips ; and =, encrypts with CBC
- `bitflip_check_admin()` — decrypts and checks for ";admin=true;"
- `cbc_bitflip_attack()` — flips 3 bytes in the ciphertext to inject the target string

## Reuses from
- `aes_128_cbc_encrypt()` / `aes_128_cbc_decrypt()` from ch10

## Summary
Even though the server sanitizes our input, CBC lets us modify the plaintext by flipping bits in the ciphertext. This is why integrity checks (like HMAC) are needed alongside encryption.
